## 🏗️ Architecture

```
Internet
   │
   ▼
AWS NLB (LoadBalancer)
   │  port 80
   ▼
banking-frontend (nginx)         ← Deployment, 1 replica
   │  /api/* → strips /api → proxy_pass :5000
   ▼
banking-app-backend (Node.js)    ← ClusterIP Service name nginx expects
   │  /api/accounts
   │  /api/transactions
   ▼
postgres-0 (PostgreSQL 16)       ← StatefulSet, 1 replica
   │  EBS gp2 volume (5Gi)
   ▼
AWS EBS PersistentVolume
```

> ⚠️ **Critical:** The nginx config inside the frontend image proxies to the hostname
> `banking-app-backend`. Your Kubernetes Service in `04-backend.yaml` **must** be named
> `banking-app-backend` (not `banking-backend`). Mismatched names cause an immediate
> nginx `[emerg] host not found in upstream` crash and `CrashLoopBackOff`.

## 💾 EBS CSI Driver Setup (Critical for PersistentVolumes)

### Why This Is Needed

Modern EKS requires the EBS CSI driver to provision EBS volumes for PersistentVolumeClaims.
Without it, the postgres PVC stays `Pending` forever and postgres never starts.

### Step 1 — Install the EBS CSI addon

```bash
aws eks create-addon \
  --cluster-name banking-eks-cluster \
  --addon-name aws-ebs-csi-driver \
  --resolve-conflicts OVERWRITE \
  --region ap-south-1
```

### Step 2 — Associate the OIDC provider

> ⚠️ This must be done before creating the IAM role, or the trust policy will be invalid.

```bash
eksctl utils associate-iam-oidc-provider \
  --cluster banking-eks-cluster \
  --region ap-south-1 \
  --approve
```

### Step 3 — Get your OIDC hash

```bash
aws eks describe-cluster \
  --name banking-eks-cluster \
  --region ap-south-1 \
  --query "cluster.identity.oidc.issuer" \
  --output text
# Example output:
# https://oidc.eks.ap-south-1.amazonaws.com/id/D0889E8CF4169C4121C292076C80B89F
# Copy only the hash at the end
```

### Step 4 — Create the IAM role manually

> ℹ️ `eksctl create iamserviceaccount` may fail with a CloudFormation error if the role already
> exists. Creating the role directly with `aws iam` is more reliable.

```bash
ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)
OIDC_ID="<paste-your-hash-here>"

cat > /tmp/ebs-trust-policy.json << EOF
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Federated": "arn:aws:iam::${ACCOUNT_ID}:oidc-provider/oidc.eks.ap-south-1.amazonaws.com/id/${OIDC_ID}"
      },
      "Action": "sts:AssumeRoleWithWebIdentity",
      "Condition": {
        "StringEquals": {
          "oidc.eks.ap-south-1.amazonaws.com/id/${OIDC_ID}:sub": "system:serviceaccount:kube-system:ebs-csi-controller-sa",
          "oidc.eks.ap-south-1.amazonaws.com/id/${OIDC_ID}:aud": "sts.amazonaws.com"
        }
      }
    }
  ]
}
EOF

# Create the IAM role
aws iam create-role \
  --role-name AmazonEKS_EBS_CSI_DriverRole \
  --assume-role-policy-document file:///tmp/ebs-trust-policy.json

# Attach the EBS policy
aws iam attach-role-policy \
  --role-name AmazonEKS_EBS_CSI_DriverRole \
  --policy-arn arn:aws:iam::aws:policy/service-role/AmazonEBSCSIDriverPolicy
```

### Step 5 — Annotate the service account

```bash
kubectl annotate serviceaccount ebs-csi-controller-sa \
  -n kube-system \
  eks.amazonaws.com/role-arn=arn:aws:iam::${ACCOUNT_ID}:role/AmazonEKS_EBS_CSI_DriverRole \
  --overwrite
```

### Step 6 — Restart the CSI controller

```bash
kubectl rollout restart deployment/ebs-csi-controller -n kube-system

# Watch until all pods show 6/6 Running
kubectl get pods -n kube-system -l app=ebs-csi-controller -w
```

Expected output:
```
NAME                                  READY   STATUS    RESTARTS   AGE
ebs-csi-controller-67c6d46667-mv57d   6/6     Running   0          50s
ebs-csi-controller-67c6d46667-xbznf   6/6     Running   0          50s
```

## ⚙️ Kubernetes Manifest Notes

### Critical: Backend Service Name Must Match Nginx Config

The frontend image's nginx config (at `/etc/nginx/conf.d/default.conf`) proxies API calls to
the hostname `banking-app-backend`. Your backend Service in `04-backend.yaml` **must** use
that exact name:

```yaml
# 04-backend.yaml — Service section
apiVersion: v1
kind: Service
metadata:
  name: banking-app-backend    # ← MUST be this, not "banking-backend"
  namespace: banking
spec:
  selector:
    app: banking-backend
  ports:
    - name: http
      port: 5000
      targetPort: 5000
  type: ClusterIP
```

You can verify what hostname nginx expects at any time by running a temporary pod:

```bash
kubectl run tmp-frontend --image=shivaram1918/banking-frontend:latest \
  --restart=Never -n banking --command -- sleep 3600

kubectl exec -it tmp-frontend -n banking -- cat /etc/nginx/conf.d/default.conf
# Look for: proxy_pass http://<hostname>:5000;
# Your Service name must match <hostname>

kubectl delete pod tmp-frontend -n banking
```

## 🗄️ Database Setup

### Why Tables Are Not Auto-Created

The backend has no migration runner. Tables must be created manually once postgres is running.

### Connect to PostgreSQL

```bash
kubectl exec -it -n banking postgres-0 -- psql -U bankuser -d bankingdb
```

### Create Tables (run inside psql)

```sql
-- Enable UUID extension
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- Accounts table
CREATE TABLE IF NOT EXISTS accounts (
  id             UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  account_number VARCHAR(20) UNIQUE NOT NULL DEFAULT '',
  full_name      VARCHAR(100) NOT NULL,
  email          VARCHAR(100) UNIQUE NOT NULL,
  phone          VARCHAR(20),
  account_type   VARCHAR(20) DEFAULT 'SAVINGS',
  balance        DECIMAL(15,2) DEFAULT 0.00,
  status         VARCHAR(20) DEFAULT 'ACTIVE',
  created_at     TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at     TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Auto-generate account number on insert
CREATE OR REPLACE FUNCTION generate_account_number()
RETURNS TRIGGER AS $$
BEGIN
  NEW.account_number := 'ACC' || LPAD(FLOOR(RANDOM() * 9000000 + 1000000)::TEXT, 7, '0');
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER set_account_number
BEFORE INSERT ON accounts
FOR EACH ROW
WHEN (NEW.account_number = '' OR NEW.account_number IS NULL)
EXECUTE FUNCTION generate_account_number();

-- Transactions table
CREATE TABLE IF NOT EXISTS transactions (
  id           UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  account_id   UUID REFERENCES accounts(id),
  type         VARCHAR(20) NOT NULL,
  amount       DECIMAL(15,2) NOT NULL,
  description  TEXT,
  balance_after DECIMAL(15,2),
  created_at   TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Verify
\dt
```

### Exit psql

```bash
\q
```

## 🔍 Database Query Commands

```bash
# All accounts
kubectl exec -n banking postgres-0 -- psql -U bankuser -d bankingdb -c \
  "SELECT account_number, full_name, email, balance, status FROM accounts ORDER BY created_at DESC;"

# All transactions
kubectl exec -n banking postgres-0 -- psql -U bankuser -d bankingdb -c \
  "SELECT a.account_number, a.full_name, t.type, t.amount, t.created_at
   FROM transactions t JOIN accounts a ON t.account_id = a.id ORDER BY t.created_at DESC;"
```

Inside an interactive psql session:

```sql
-- List all tables
\dt

-- Describe table structure
\d accounts
\d transactions

-- Account summary
SELECT COUNT(*) as total_accounts, SUM(balance) as total_balance, AVG(balance) as avg_balance
FROM accounts;

-- Transaction summary by type
SELECT type, COUNT(*) as count, SUM(amount) as total FROM transactions GROUP BY type;

-- Check specific account
SELECT * FROM accounts WHERE email = 'your@email.com';

-- Recent transactions for an account
SELECT t.*
FROM transactions t
JOIN accounts a ON t.account_id = a.id
WHERE a.email = 'your@email.com'
ORDER BY t.created_at DESC LIMIT 10;
```

## 🚨 Troubleshooting

### ❌ Frontend: `CrashLoopBackOff` — `host not found in upstream`

```
[emerg] 1#1: host not found in upstream "banking-app-backend" in /etc/nginx/conf.d/default.conf:33
```

**Cause:** The nginx config inside the Docker image resolves the backend by the Kubernetes Service
name. If that Service doesn't exist, nginx refuses to start.

**Fix:** Ensure the backend Service in `04-backend.yaml` is named `banking-app-backend`:

```bash
kubectl get svc -n banking        # confirm the service name
kubectl apply -f 04-backend.yaml  # apply after renaming
kubectl rollout restart deployment/banking-frontend -n banking
```

---

### ❌ EBS CSI Controller: `CrashLoopBackOff` after install

**Cause:** The EBS CSI controller pod can't assume the IAM role because:
- The OIDC provider wasn't associated before the role was created, or
- The service account wasn't annotated with the role ARN.

**Fix — full sequence:**

```bash
# 1. Associate OIDC provider (if not done yet)
eksctl utils associate-iam-oidc-provider \
  --cluster banking-eks-cluster --region ap-south-1 --approve

# 2. Re-annotate the service account with the correct role ARN
kubectl annotate sa ebs-csi-controller-sa -n kube-system \
  eks.amazonaws.com/role-arn=arn:aws:iam::<ACCOUNT_ID>:role/AmazonEKS_EBS_CSI_DriverRole \
  --overwrite

# 3. Restart the controller
kubectl rollout restart deployment/ebs-csi-controller -n kube-system

# 4. Verify — should reach 6/6 Running
kubectl get pods -n kube-system -l app=ebs-csi-controller -w
```

> ⚠️ `eksctl create iamserviceaccount` may fail with a CloudFormation error if the role already
> exists. Skip it and annotate the service account manually as shown above.

---

### ❌ `eksctl create iamserviceaccount` fails with CloudFormation error

**Cause:** The role `AmazonEKS_EBS_CSI_DriverRole` already exists (created manually), so
CloudFormation tries and fails to create it again.

**Fix:** Skip `eksctl create iamserviceaccount` entirely. The role is already in place.
Just annotate the existing service account:

```bash
kubectl annotate sa ebs-csi-controller-sa -n kube-system \
  eks.amazonaws.com/role-arn=arn:aws:iam::<ACCOUNT_ID>:role/AmazonEKS_EBS_CSI_DriverRole \
  --overwrite
kubectl rollout restart deployment/ebs-csi-controller -n kube-system
```

---

### ❌ `kubectl apply` — `the path "k8s/..." does not exist`

**Cause:** You are running the command from inside the `k8s/` directory, so the relative
path `k8s/04-backend.yaml` doesn't resolve.

**Fix:** Always run `kubectl apply` from the project root, or drop the `k8s/` prefix:

```bash
# From project root:
cd /home/ubuntu/simple_Project
kubectl apply -f k8s/04-backend.yaml

# OR from inside k8s/:
cd /home/ubuntu/simple_Project/k8s
kubectl apply -f 04-backend.yaml
```

---

### ❌ Postgres Pod Pending

```bash
kubectl describe pvc postgres-storage-postgres-0 -n banking
kubectl get events -n banking
```

Common causes:
- EBS CSI driver not running → fix IAM role and restart controller (see above)
- Wrong `storageClassName` → must match: `kubectl get storageclass`
- Node capacity full → `kubectl describe nodes | grep -E "memory|Taints"`

---

### ❌ Backend CrashLoopBackOff

```bash
kubectl logs -n banking deployment/banking-backend --tail=30
```

Common causes:
- Postgres not ready → the `initContainer` `wait-for-postgres` handles this automatically
- Tables not created → run the `CREATE TABLE` commands in the Database Setup section
- Wrong DB credentials → check configmap and secret values

---

### ❌ 500 Error on API calls

```bash
kubectl logs -n banking deployment/banking-backend -f
```

Common causes:
- Tables don't exist → run `CREATE TABLE` commands
- Duplicate empty `account_number` → create the trigger (see Database Setup)
- Wrong env vars → `kubectl get configmap banking-config -n banking -o yaml`
